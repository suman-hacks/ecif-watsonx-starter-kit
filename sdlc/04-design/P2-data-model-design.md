## Migration Plan (Brownfield / Mainframe Modernization)

### Phase 1: Dual-Write Setup
- Application writes to both legacy and new schema simultaneously
- New schema reads do not affect production — shadow mode
- Duration: [Sprint N — Sprint N]

### Phase 2: Read Switch
- New schema is now the read source for [services listed]
- Legacy schema still receives writes — fallback still possible
- Validation: reconciliation job runs nightly, flags discrepancies > 0.01%
- Duration: [Sprint N — Sprint N]

### Phase 3: Legacy Cutoff
- Legacy schema writes stopped
- Legacy schema retained read-only for [N months] for rollback
- Duration: [Sprint N]

### Rollback Trigger
Roll back to Phase 1 if:
- Data discrepancy > 0.01% in reconciliation job
- Any BLOCKER data integrity issue confirmed
- Performance degradation > 20% vs baseline
