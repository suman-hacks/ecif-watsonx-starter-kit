# P2: Infrastructure as Code Generation

## When to Use This Prompt
During Stage 08 (Deployment), after the deployment plan (P1) is approved. Use to generate IaC templates for the target infrastructure — Kubernetes manifests, Helm charts, Terraform modules, or CDK stacks.

---

## Prompt

```text
You are a FORGE DevSecOps engineer generating infrastructure as code. All generated IaC must follow security-by-default principles and be production-grade.

INPUTS
Cloud platform: [AWS / Azure / GCP / On-premise Kubernetes / IBM Cloud]
IaC tool: [Terraform / AWS CDK / Helm + Kubernetes / Ansible / OpenShift]
Services to provision: [List services from Agent 04 service map]
NFRs: [Paste NFRs from Stage 02 — scalability, availability, performance targets]
Security requirements: [Paste from P1 threat model and P3 compliance review]
Existing infrastructure: [Describe existing VPC/network, or "Greenfield — generate from scratch"]

RULES FOR IaC GENERATION
- No hardcoded credentials, API keys, or connection strings in any file
- All secrets from vault or cloud-native secret manager (AWS Secrets Manager / Azure Key Vault / HashiCorp Vault)
- All resources tagged: environment, project, owner, cost-center, compliance-scope
- Least-privilege IAM: services get only the permissions they need — no Admin policies
- Network: private subnets for application and data tiers; public only for load balancers
- Encryption: at rest and in transit for all data stores
- Autoscaling configured for all stateless services

TASK
Generate complete IaC for the [SERVICE-NAME] service.

## Kubernetes Deployment (Helm Chart Structure)

### Chart layout
```
helm/[service-name]/
├── Chart.yaml
├── values.yaml              # Default values — no environment specifics
├── values-dev.yaml          # Dev environment overrides
├── values-staging.yaml      # Staging overrides
├── values-prod.yaml         # Production overrides
└── templates/
    ├── deployment.yaml
    ├── service.yaml
    ├── ingress.yaml
    ├── hpa.yaml              # Horizontal Pod Autoscaler
    ├── pdb.yaml              # Pod Disruption Budget
    ├── configmap.yaml
    ├── secret.yaml           # References to external secret (not values)
    ├── serviceaccount.yaml
    └── networkpolicy.yaml
```

### Chart.yaml
```yaml
apiVersion: v2
name: authorization-service
description: FORGE — Authorization Service Helm Chart
type: application
version: 0.1.0
appVersion: "1.0.0"
```

### values.yaml
```yaml
replicaCount: 2  # Minimum for HA — prod overrides to 3+

image:
  repository: [registry]/authorization-service
  pullPolicy: IfNotPresent
  tag: ""  # Overridden by CI/CD pipeline

service:
  type: ClusterIP
  port: 8080
  targetPort: 8080

resources:
  requests:
    cpu: "500m"
    memory: "512Mi"
  limits:
    cpu: "1000m"
    memory: "1Gi"

autoscaling:
  enabled: true
  minReplicas: 2
  maxReplicas: 10
  targetCPUUtilizationPercentage: 70
  targetMemoryUtilizationPercentage: 80

# No credentials here — sourced from external secret manager
externalSecrets:
  enabled: true
  secretStoreName: "aws-secrets-manager"  # or "azure-keyvault"
  secrets:
    - name: db-credentials
      remoteKey: "prod/authorization-service/db"
    - name: fraud-api-key
      remoteKey: "prod/authorization-service/fraud-api"

# Health probes
livenessProbe:
  httpGet:
    path: /actuator/health/liveness
    port: 8080
  initialDelaySeconds: 30
  periodSeconds: 10
  failureThreshold: 3

readinessProbe:
  httpGet:
    path: /actuator/health/readiness
    port: 8080
  initialDelaySeconds: 20
  periodSeconds: 5
  failureThreshold: 3

# Pod disruption budget — keep at least 1 pod during updates
podDisruptionBudget:
  enabled: true
  minAvailable: 1
```

### templates/deployment.yaml
```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: {{ include "authorization-service.fullname" . }}
  labels:
    {{- include "authorization-service.labels" . | nindent 4 }}
    app.kubernetes.io/component: authorization
    compliance-scope: pci-dss
spec:
  replicas: {{ .Values.replicaCount }}
  selector:
    matchLabels:
      {{- include "authorization-service.selectorLabels" . | nindent 6 }}
  strategy:
    type: RollingUpdate
    rollingUpdate:
      maxSurge: 1
      maxUnavailable: 0  # Zero-downtime: never kill a pod before a new one is ready
  template:
    metadata:
      labels:
        {{- include "authorization-service.selectorLabels" . | nindent 8 }}
      annotations:
        prometheus.io/scrape: "true"
        prometheus.io/port: "8080"
        prometheus.io/path: "/actuator/prometheus"
    spec:
      serviceAccountName: {{ include "authorization-service.serviceAccountName" . }}
      securityContext:
        runAsNonRoot: true
        runAsUser: 1000
        fsGroup: 1000
        seccompProfile:
          type: RuntimeDefault
      containers:
        - name: {{ .Chart.Name }}
          image: "{{ .Values.image.repository }}:{{ .Values.image.tag | default .Chart.AppVersion }}"
          imagePullPolicy: {{ .Values.image.pullPolicy }}
          securityContext:
            allowPrivilegeEscalation: false
            readOnlyRootFilesystem: true
            capabilities:
              drop:
                - ALL
          ports:
            - name: http
              containerPort: 8080
              protocol: TCP
          env:
            - name: SPRING_PROFILES_ACTIVE
              value: {{ .Values.environment | default "prod" }}
            - name: DB_URL
              valueFrom:
                secretKeyRef:
                  name: db-credentials
                  key: url
            - name: DB_USERNAME
              valueFrom:
                secretKeyRef:
                  name: db-credentials
                  key: username
            - name: DB_PASSWORD
              valueFrom:
                secretKeyRef:
                  name: db-credentials
                  key: password
          resources:
            {{- toYaml .Values.resources | nindent 12 }}
          livenessProbe:
            {{- toYaml .Values.livenessProbe | nindent 12 }}
          readinessProbe:
            {{- toYaml .Values.readinessProbe | nindent 12 }}
          volumeMounts:
            - name: tmp
              mountPath: /tmp  # Required since readOnlyRootFilesystem: true
      volumes:
        - name: tmp
          emptyDir: {}
      affinity:
        podAntiAffinity:
          preferredDuringSchedulingIgnoredDuringExecution:
            - weight: 100
              podAffinityTerm:
                labelSelector:
                  matchLabels:
                    {{- include "authorization-service.selectorLabels" . | nindent 20 }}
                topologyKey: kubernetes.io/hostname  # Spread across nodes
```

### templates/networkpolicy.yaml
```yaml
# Deny all by default; allow only what's needed
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: {{ include "authorization-service.fullname" . }}-network-policy
spec:
  podSelector:
    matchLabels:
      {{- include "authorization-service.selectorLabels" . | nindent 6 }}
  policyTypes:
    - Ingress
    - Egress
  ingress:
    - from:
        - namespaceSelector:
            matchLabels:
              name: ingress-nginx  # Allow only from ingress controller
      ports:
        - protocol: TCP
          port: 8080
  egress:
    - to:
        - namespaceSelector:
            matchLabels:
              name: database  # Allow to database namespace
      ports:
        - protocol: TCP
          port: 5432  # PostgreSQL
    - to:
        - namespaceSelector:
            matchLabels:
              name: kafka  # Allow to Kafka namespace
      ports:
        - protocol: TCP
          port: 9092
    - ports:
        - protocol: UDP
          port: 53  # DNS
```

## Terraform: RDS PostgreSQL (AWS)

```hcl
# rds.tf — PostgreSQL for authorization-service
resource "aws_db_instance" "authorization_db" {
  identifier = "${var.environment}-authorization-db"
  
  engine         = "postgres"
  engine_version = "15.4"
  instance_class = var.db_instance_class  # e.g., db.r6g.large for prod
  
  allocated_storage     = var.db_storage_gb
  max_allocated_storage = var.db_max_storage_gb  # Storage autoscaling
  storage_type          = "gp3"
  storage_encrypted     = true
  kms_key_id            = aws_kms_key.authorization_db_key.arn
  
  db_name  = "authorizationdb"
  username = "authsvc"
  password = random_password.db_password.result  # Stored in Secrets Manager, not here
  
  vpc_security_group_ids = [aws_security_group.authorization_db.id]
  db_subnet_group_name   = aws_db_subnet_group.authorization.name
  
  # High availability
  multi_az = var.environment == "prod" ? true : false
  
  # Backup
  backup_retention_period = var.environment == "prod" ? 7 : 1
  backup_window          = "03:00-04:00"  # UTC
  maintenance_window     = "Mon:04:00-Mon:05:00"
  
  # Deletion protection
  deletion_protection = var.environment == "prod" ? true : false
  skip_final_snapshot = var.environment == "prod" ? false : true
  
  # Enhanced monitoring
  monitoring_interval = 60
  monitoring_role_arn = aws_iam_role.rds_monitoring.arn
  
  # Performance Insights
  performance_insights_enabled          = true
  performance_insights_retention_period = 7
  
  tags = local.common_tags
}

# Security group — only allow from application tier
resource "aws_security_group" "authorization_db" {
  name_prefix = "${var.environment}-authorization-db-"
  vpc_id      = var.vpc_id
  
  ingress {
    from_port       = 5432
    to_port         = 5432
    protocol        = "tcp"
    security_groups = [aws_security_group.authorization_app.id]  # App tier only
    description     = "PostgreSQL from authorization-service"
  }
  
  egress {
    from_port   = 0
    to_port     = 0
    protocol    = "-1"
    cidr_blocks = ["0.0.0.0/0"]
  }
  
  tags = local.common_tags
}

# Store credentials in Secrets Manager — never in state file as plaintext
resource "aws_secretsmanager_secret" "db_credentials" {
  name                    = "/${var.environment}/authorization-service/db"
  recovery_window_in_days = 7
  tags                    = local.common_tags
}

resource "aws_secretsmanager_secret_version" "db_credentials" {
  secret_id = aws_secretsmanager_secret.db_credentials.id
  secret_string = jsonencode({
    url      = "jdbc:postgresql://${aws_db_instance.authorization_db.endpoint}/authorizationdb"
    username = aws_db_instance.authorization_db.username
    password = random_password.db_password.result
  })
}
```

## IaC Security Checklist

Before applying to any environment:
- [ ] No hardcoded credentials, keys, or connection strings in any .tf or values.yaml file
- [ ] All secrets sourced from external secret manager
- [ ] Network policies restricting traffic to minimum required
- [ ] Encryption at rest enabled for all storage
- [ ] Encryption in transit enforced (TLS)
- [ ] IAM roles use least-privilege permissions
- [ ] Deletion protection enabled in production
- [ ] All resources tagged with compliance scope
- [ ] `terraform plan` output reviewed by Lead Engineer before `terraform apply`
- [ ] IaC changes go through same PR process as application code
```
