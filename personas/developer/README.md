# Persona: Developer (Engineer)

## Role Overview

Developers use FORGE to accelerate implementation while maintaining quality, security, and traceability standards. FORGE prompts ensure AI-generated code is grounded in approved design artifacts — not invented from a verbal description.

## What AI Can Help You With

- Generating implementation classes and test suites from design specs
- AI-assisted code reviews (before submitting PRs)
- Debugging root cause analysis from logs and stack traces
- Refactoring legacy code to clean patterns
- Generating documentation and PR descriptions
- Understanding legacy COBOL or old Java you've inherited

## FORGE Rules for Developers

1. **Never generate code without a design spec** — if Stage 03/04 isn't done, escalate
2. **Always generate tests alongside code** — no exceptions
3. **Document all assumptions** — anything not in the spec goes in an Assumptions section
4. **One PR per feature** — link to the user story and business rules it implements
5. **Run the code review prompt on yourself** before submitting to a human

## SDLC Stages You Contribute To

- Stage 04 (Design) — detailed implementation planning
- Stage 05 (Development) — primary contributor
- Stage 06 (Testing) — unit and integration tests
