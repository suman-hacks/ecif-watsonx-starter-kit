# Persona: DevSecOps Engineer

## Role Overview

DevSecOps Engineers use FORGE to embed security at every stage of delivery — from threat modeling before architecture is finalized, through security code review, to CI/CD pipeline hardening and production security monitoring.

## What AI Can Help You With

- Generating STRIDE threat models from architecture diagrams
- Security-focused code reviews (OWASP Top 10 analysis)
- Generating CI/CD pipeline configurations with security gates
- Infrastructure as Code (Terraform/CDK/Helm) generation and review
- Compliance gap assessments (PCI-DSS, HIPAA, GDPR, SOX)
- Secrets management design
- Security incident response runbooks

## FORGE Rules for DevSecOps

1. Every architecture must have a threat model before code is written
2. Security findings rated BLOCKER prevent merge — no exceptions without CISO approval
3. No secrets in code, ever — use approved secrets manager
4. All IaC must include security controls: encryption, least-privilege IAM, network segmentation
5. Compliance requirements must be verifiable in the CI/CD pipeline, not just in documentation

## SDLC Stages You Lead

- Stage 07 (Security) — primary driver
- Stage 03 (Architecture) — security architecture reviewer
- Stage 08 (Deployment) — pipeline and infrastructure security
- Stage 09 (Operations) — security monitoring and incident response
