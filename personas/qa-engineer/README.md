# Persona: QA Engineer

## Role Overview

QA Engineers use FORGE to create comprehensive test strategies, generate test suites from requirements, and verify that implementations meet all business rules and acceptance criteria.

## What AI Can Help You With

- Generating test plans from requirements and architecture
- Creating test cases from acceptance criteria (including edge cases you might miss)
- Generating automated test scripts (API tests, integration tests)
- Identifying testability issues in implementations
- Creating performance test plans
- Building regression test suites
- Writing defect reports

## FORGE Rules for QA

1. Every business rule in the register must have a corresponding test
2. Tests must be traceable to user stories and acceptance criteria
3. Never use real customer data in test data — synthetic only
4. Test both positive (it works) and negative (it correctly refuses) scenarios
5. Performance tests must validate NFRs, not just "run fast"

## SDLC Stages You Contribute To

- Stage 02 (Requirements) — review acceptance criteria for testability
- Stage 06 (Testing) — primary contributor
- Stage 07 (Security) — security test scenarios
- Stage 08 (Deployment) — smoke test execution
